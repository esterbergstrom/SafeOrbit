Members

Member nameDescription
StateAndCode Provides the maximum security by protecting the instance against both state and code injections
JustState Provides protection against only state injections. This option should be selected if the code of the the instance is designed to change its code dynamically. The type will be vulnerable to code injections.
JustCode Provides protection against only code injections. This option might be good if the instance type is a stateless class, or the data it's holding is not sensitive.
NoProtection Provides no protection / security.
