Remarks

This is very similar to a MemoryStream, except, once data has been read from the SafeMemoryStream, it disappears. You cannot seek, you cannot get or set position. If you write to the SafeMemoryStream, the data is written to the end, and and when you read, the beginning moves closer to the end.

Length tells you how many bytes are currently in memory.

After calling Close, the SafeMemoryStream cannot be written to anymore, but it can still be read from.

The writer should call Close when it's done writing. The reader may optionally call Close when it's done reading.

After Close has been called, the reader may read up to the number of bytes available, and subsequent calls to Read(array<Byte>[]()[][], Int32, Int32) will return 0. Read(array<Byte>[]()[][], Int32, Int32) will never return 0, until after Close has been called, and all the bytes have been read.
