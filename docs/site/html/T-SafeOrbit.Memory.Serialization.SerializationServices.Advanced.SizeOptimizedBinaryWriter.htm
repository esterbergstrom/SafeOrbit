SizeOptimizedBinaryWriter Class

Stores data in a binary format. Data is stored in two steps. At first are all objects stored in a cache and all types are analyzed. Then all types and property names are sorted and placed in a list. Duplicates are removed. Serialized objects contain references to these types and property names. It decreases file size, especially for serialization of collection (many items of the same type) SizeOptimizedBinaryWriter has bigger overhead than BurstBinaryWriter
